Using Variables

Sometimes you may need to replace part of an URL or construct new. In LinCastor Browser you can do this either by writing a plugin extension or by creating new URL from variables. You can use any variable that an extension or rules could use.

URL          => my-http://myhost.domain.com:8080/mysite/a.html?search=blah#myanchor
URL_SCHEME   => my-http
URL_HOST     => myhost.domain.com
URL_PORT     => 8080
URL_PATH     => /mysite/a.html
URL_PATH_EXT  => html
...

For complete list of variables see here.

NOTE: variables are not encoded by default

Use X_ variable prefix to return encoded value. This is very useful if you want to pass value as part of URL’s query string.

Using **** placeholder

The **** placeholder text will be replaced by any selected text. This is equivalent of using ${TEXT} variable. This makes a lot of sense for lookup URLs where you want to pass current text selection to a website (e.g. google search).

Replacing part of URL

Some apps are no able to handle standard HTTP or HTTPS links directly and use custom schemes instead. For example, Zoom app needs zoommtg. You can construct new URL by replacing URL_SCHEME (“https") by value “zoommtg" and passing it to the Zoom app to open it.
